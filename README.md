# PDF 작업대 (PDF Workbench)

브라우저에서 바로 쓰는 PDF 편집 도구. **회전 · 분할 · 병합 · 텍스트 편집**을
서버 없이 클라이언트에서 처리합니다. 업로드한 파일은 어디로도 전송되지 않고
사용자의 브라우저 안에서만 다뤄집니다.

## 기능

- **열기 / 병합** — 여러 PDF를 한 번에 열면 페이지가 하나의 목록으로 이어져 병합 대기 상태가 됩니다.
- **순서 변경** — 왼쪽 썸네일을 드래그하거나 ↑/↓ 버튼으로 페이지 순서를 바꿉니다.
- **회전** — 페이지별로 90°씩 회전.
- **삭제** — 페이지 단위 삭제.
- **분할** — 체크박스로 원하는 페이지만 골라 별도 PDF로 저장.
- **텍스트 편집** — 기존 글자를 클릭해 수정하거나 빈 곳을 클릭해 새 텍스트 추가. 한글 지원.
- **내보내기** — 편집 결과를 하나의 PDF로 저장.

## 기술 스택

- Vite + React (클라이언트 전용, 서버리스)
- `pdfjs-dist` — 페이지 렌더링 및 텍스트 위치 추출
- `pdf-lib` + `@pdf-lib/fontkit` — 회전 · 병합 · 텍스트 그리기 · 폰트 임베딩
- 한글 폰트: `public/fonts/KoreanText-Regular.ttf` (나눔고딕, 편집 텍스트에 한글이 있을 때만 임베딩)

## 로컬 실행

```bash
npm install
npm run dev      # 개발 서버
npm run build    # 프로덕션 빌드 (dist/)
npm run preview  # 빌드 결과 미리보기
```

## Vercel 배포

이 저장소를 GitHub에 올린 뒤 Vercel에서 Import 하면 됩니다. 별도 설정 없이 자동 인식됩니다.

- Framework Preset: **Vite**
- Build Command: `npm run build`
- Output Directory: `dist`

CLI로 배포하려면:

```bash
npm i -g vercel
vercel        # 미리보기 배포
vercel --prod # 프로덕션 배포
```

## 텍스트 편집의 원리와 한계

PDF는 글자가 좌표에 고정된 포맷이라, 워드처럼 글이 자동으로 흐르지 않습니다.
이 도구는 상용 편집기와 동일한 방식으로 처리합니다 — **원래 글자 영역을 흰색으로 덮고
그 자리에 새 글자를 다시 그립니다.**

그래서 다음은 알고 쓰는 게 좋습니다.

- 바꾼 글자가 원래보다 길면 흰색 덮개 밖으로 삐져나올 수 있습니다(글자는 그려지되 배경만 짧게 덮임).
- 원문 폰트를 그대로 재현하지 않고, 편집 글자는 나눔고딕(한글)/Helvetica(영문)로 그립니다.
- 회전된 페이지는 좌표가 어긋나므로, 편집 전 회전을 0°로 되돌린 뒤 편집하세요(출력 시 다시 회전 가능).
- 복잡한 표 안의 텍스트나 자동 줄바꿈은 완벽히 재배치되지 않습니다.

단순 문구 교정·정정, 도장/서명 자리 텍스트, 날짜·이름 수정 같은 실무 용도에는 충분합니다.
